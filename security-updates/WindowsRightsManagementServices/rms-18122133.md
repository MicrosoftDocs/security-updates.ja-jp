RMS の信頼階層
==============

RMS システムは、Microsoft 登録サービス、組織の RMS サーバー、クライアント コンピュータ、システムのユーザーなどで構成されます。各コンポーネントには、システムにおける ID を確立するための証明書が発行されます。信頼階層は、証明書と、それを所有するエンティティ間の信頼関係を定義します。信頼されたエンティティと、そのエンティティが他の信頼されたエンティティに対して発行したライセンス間の信頼関係も定義されます。

信頼階層は、信頼チェーンとして、証明書とライセンスとを結びつけます。RMS は、特定の証明書またはライセンスを起点とし、この信頼チェーンをたどることで、信頼されたキー ペアを見つけることができます。信頼チェーンには、現在の証明書、その証明書を発行したエンティティの証明書、エンティティの証明書を発行したエンティティの証明書...というように、信頼のルートを頂点とする階層が格納されています。

RMS の場合、信頼ルート (つまり、"信頼の頂点") は、Microsoft キー ペアになります。信頼のルートは 1 つだけであるため、企業は、信頼されたエンティティ (社内外のユーザーやパートナーなど) を内包する信頼系統を確立することができます。

以下は、組織における信頼階層を示しています。信頼チェーンの頂点は、ベース証明書を発行した Microsoft サービスになります。

![](images/Cc720232.6c169175-94fb-4ec0-93bc-12748aae3ac4(WS.10).gif)
1.  各クライアント コンピュータに、Microsoft のルート公開キーを格納する、固有のロックボックスが発行されます。
2.  ライセンス要求を受け取ると、RMS が信頼ルートまで信頼階層をさかのぼり、プリンシパルを検証します。
3.  RMS が、ライセンスに登録された信頼済みエンティティの信頼性を確認します。
4.  RMS が、信頼階層に存在するサーバーによって発行された、信頼済みエンティティの証明書を検証します。

RMS は、証明書チェーンの各レベルでライセンスまたは証明書を検証し、信頼チェーンをたどって、既知の信頼ルートに到達できるかどうかを確認します。RMS は、チェーンに含まれる各ライセンスまたは証明書をチェックし、次の条件が満たされているかどうかを確認します。

-   XrML が有効であること。
-   発行者の署名が有効であること。
-   ライセンスのセマンティクスが、対象の用途に対して適切であること。
-   条件 (有効期限日など) が満たされていること。
-   ライセンスが失効されていないこと。
-   ライセンスの署名キーと、証明された発行者のキーとが一致していること。
