Microsoft Windows Server Update Services の概要
===============================================

WSUS の展開シナリオ

公開日: 2005年6月17日

WSUS は、ダイヤルアップ接続を使用する小規模な企業から、複数のサイトに数千人のユーザーが分散している大規模な企業に及ぶ、多様な組織が抱える更新プログラム管理のニーズに柔軟に対応することができます。組織の規模、所在地、およびその接続インフラストラクチャに応じて、管理者は WSUS サーバーの台数を 1 から多数の範囲で最も効率的にスケール アウトする方法を決定できます。

このセクションでは、WSUS のコンポーネントを、小規模、中規模、および制限されたネットワークに展開する場合の一般的なシナリオについて詳しく説明します。

### WSUS サーバーが 1 台の場合 (小規模または単純なネットワーク)

WSUS サーバーを 1 台だけ使用するシナリオでは、管理者が企業ファイアウォールの内部で WSUS を実行しているサーバーを設定します。この場合、次の図のように、コンテンツは Microsoft Update と直接同期され、更新プログラムがクライアント コンピュータに配布されます。

### WSUS サーバーが複数ある場合 (中規模またはより複雑なネットワーク)

中規模またはより複雑なネットワークで WSUS コンポーネントを展開する場合の一般的なシナリオは、次のとおりです。

#### 独立した複数の WSUS サーバー

次の図のように、それぞれが個別に管理され、かつそのコンテンツが Microsoft Update から同期されるように構成されている複数のサーバーを展開することができます。

![](images/Cc708628.30559d49-ce7a-483c-b0b3-7b66f479391e(ja-jp,WS.10).gif)
**独立した複数の WSUS サーバー**

このシナリオでの展開方法は、ローカル エリア ネットワーク （LAN） または広域ネットワーク (WAN) のさまざまなセグメントが、独立した事業体 (支社など) として管理されている状況に適しています。また、この方法が適しているケースとして、WSUS を実行しているあるサーバーが、特定のオペレーティング システム (Windows 2000 など) を実行しているクライアント コンピュータのみに更新プログラムを展開するように構成され、別のサーバーが別のオペレーティング システム (Windows XP など) を実行しているクライアント コンピュータのみに更新プログラムを展開するように構成されている場合も挙げられます。これらの状況では、2 台のサーバーがコンテンツを同期する必要がありません。

#### 内部で同期される複数の WSUS サーバー

WSUS を実行していて、組織のイントラネット内ですべてのコンテンツを同期する複数のサーバーを展開することができます。次の図では、1 台のサーバーのみがインターネットに公開されています。この構成では、このサーバーのみが Microsoft Update から更新プログラムをダウンロードします。このサーバーはアップストリーム サーバーとして設定され、これをソースとしてダウンストリーム サーバーが同期されます。必要であれば、地理的に分散するネットワーク全体にサーバーを配置して、すべてのクライアント コンピュータに最適な接続を提供することができます。

![](images/Cc708628.3bdd2c72-270a-4109-9703-06adc6467061(ja-jp,WS.10).gif)
**内部で同期される複数の WSUS サーバー**

![](images/Cc708628.arrow_px_up(ja-jp,WS.10).gif) [ページのトップへ](#ctl00_rs1_eb1_panel1)

### WSUS サーバーが接続されていない場合 (制限または制約されたインターネット接続)

企業ポリシーまたはその他の条件によってインターネットへのコンピュータ アクセスが制限される場合は、WSUS を実行している内部サーバーを次の図のように設定することができます。この例では、インターネットに接続され、かつイントラネットからは切り離されているサーバーが作成されています。このサーバーに更新プログラムをダウンロードし、これをテストして承認した後で、管理者は更新プログラムのメタデータとコンテンツを CD にエクスポートします。さらに、イントラネット内の WSUS を実行しているサーバーに、この CD から更新プログラムのメタデータとコンテンツをインポートします。次の図では、このモデルが最も単純な形で示されていますが、どのような規模の展開にも合わせることができます。

**イントラネットがインターネットに接続されていない状況での、接続されていない WSUS サーバー**
