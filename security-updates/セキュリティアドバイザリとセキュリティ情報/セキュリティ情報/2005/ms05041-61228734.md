脆弱性の詳細
------------

<span></span>
#### リモート デスクトップ プロトコル の脆弱性 - CAN-2005-1218:

[サービス拒否](http://www.microsoft.com/japan/security/glossary.mspx) の脆弱性が存在し、これにより攻撃者が特別な細工をしたリモート データ プロトコル (RDP) メッセージを、影響を受けるコンピュータに送る可能性があります。 攻撃者はコンピュータの応答を停止させる可能性があります。

#### 「リモート デスクトップ プロトコル の脆弱性」の問題を緩和する要素 - CAN-2005-1218:

-   ファイアウォールによる最善策および標準の既定のファイアウォールの構成により、ネットワーク境界の外部から行われる攻撃からネットワークを保護することができます。インターネットに接続したコンピュータでは、最低限の数のポートを開くようにすることが最善策として推奨されます。
-   リモート デスクトップ プロトコール (RDP) はどのオペレーティング システムのバージョンでも既定で有効ではありません。 Windows XP および Windows Server 2003 で、リモート アシスタンスは RDP を有効にできます。 Windows XP Media Center Edition では、Media Center Extender がインストールされている場合は、RDP を有効にできます。 Media Center Extenders に関する詳細情報は、次の [Web サイト](http://msdn2.microsoft.com/en-us/library/ms123402.aspx) (英語情報) をご覧ください。

    Small Business Server 2000 および Windows Small Business Server 2003 では、RDP は既定で有効になっています。 しかし、Windows Small Business Server 2003 およびそれ以前のバージョンは、既定で、RDP のプロトコルのコミュニケーション ポートがインターネットからブロックされています。そして RDP は、Terminal Services または Remote Web Workplace の機能が電子メールとインターネット接続の構成ウィザード (CEICW) を使用して有効にされていない限り、ローカル ネットワークでのみ利用可能です。

-   リモート デスクトップが手動で有効にされている場合は、オペレーティング システムのバージョンに応じて、以下の Windows ファイアウォールの変更が行なわれます。
    -   Windows ファイアウォールが有効にされた、Windows XP Service Pack 2 のコンピュータでは、リモート デスクトップ機能を有効にすると、「任意のコンピュータ (インターネット上のコンピュータを含む)」のスコープで、ファイアウォールでリモート デスクトップの例外が自動的に有効にされます。 リモート デスクトップを無効にした場合、このファイアウォールの例外は自動的に無効になります。
    -   Windows XP Service Pack 1、Windows Server 2003、および Windows Server 2003 Service Pack 1 では、リモート デスクトップの機能を有効にしても、ファイアウォールのリモート デスクトップの例外は有効になりません。 リモート デスクトップを有効にすると、この例外を手動で有効にするように、ダイアログ ボックスが表示します。 ファイアウォールの例外の一覧の中には、ユーザーが手動で有効にする必要のあるリモート デスクトップのエントリがあります。 リモート デスクトップを無効にしても、ファイアウォールの例外な状態に変更はありません。 コンピュータはリモート デスクトップによる問題に対しては、影響を受けなくなりますが、依然としてリモート アシスタンスおよびターミナル サービスが利用可能である場合、脆弱性の影響を受ける可能性があります。

#### 「リモート デスクトップ プロトコル の脆弱性」の回避策 - CAN-2005-1218:

マイクロソフトは次の回避策のテストを行ないました。これらの回避策は根本的な脆弱性を修正しませんが、既知の攻撃の方法を阻止する手助けとなります。回避策が機能の低下の原因となる場合、下記に示します。

-   **すでに必要がない場合、ターミナルサービス、リモートデスクトップ、リモートアシスタンス、および** **Windows Small Business Server 2003** **のリモート** **Web** **ワークプレースの機能を無効にします。**
    -   これらのサービスがすでにコンピュータに必要なければ、セキュリティの最善策としてこれらを無効にすることを検討してください。 使用しない不要なサービスを無効にすることは、セキュリティ上の脆弱性の影響を軽減する手助けになります。
    -   手動でリモート デスクトップを無効にする方法の詳細については、以下の [Web サイト](http://www.microsoft.com/resources/documentation/windows/xp/all/proddocs/en-us/rdesktop_disable.mspx) (英語情報) をご覧ください。
    -   グループ ポリシーを使用してリモート デスクトップを無効にする方法については、サポート技術情報 [306300](http://support.microsoft.com/kb/306300) をご覧ください
    -   手動で、およびグループ ポリシーを使用してリモート アシスタンスを無効にする方法を含む、リモート アシスタンスに関する情報は [Web サイト](http://www.microsoft.com/technet/prodtechnol/winxppro/maintain/intmgmt/20_xprem.mspx) (英語情報) をご覧ください。
    -   Windows Small Business Server 2003 Terminal Services およびリモート Web ワークプレースの機能を無効にする方法に関する詳細は、次の [Web サイト](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) (英語情報) をご覧ください。
-   **エンタープライズの境界領域のファイアウォールで、TCP** **ポート** **3389** **をブロックします。**

    これらのポートは、影響を受けるコンポーネントとの接続を開始するために使用されます。 ネットワークの境界領域のファイアウォールでこれらのポートをブロックすると、ファイアウォールの背後のコンピュータを、この脆弱性を悪用した攻撃から保護することができます。 これにより、エンタープライズの境界領域外で発生した攻撃からネットワークを防御する手助けとなります。 エンタープライズの境界領域で影響を受けるポートをブロックすることは、インターネットをベースにした攻撃を避ける手助けとなる最善の防御です。 しかし、エンタープライズの境界領域内からの攻撃に対しては、コンピュータは依然として影響を受ける可能性があります。 さらに、Windows XP および Windows Server 2003 にある Windows ファイアウォールは個々のコンピュータを保護する手助けとなります。 Windows ファイアウォールはWindows XP Service Pack 2 でリモート デスクトップの機能が有効にされた場合を除いて、既定では、このポートへの接続を許可しません。 これらのプラットフォーム上のリモート デスクトップの Windows ファイアウォールの例外を無効にする方法については、次の[Web サイト](http://technet2.microsoft.com/windowsserver/en/library/8b5e3b52-b77b-4d98-a058-b8083f3935831033.mspx) (英語情報) をご覧ください。

    リモート デスクトップの Windows ファイアウォールの例外を無効にできない場合、ローカル ネットワークに \[任意のコンピュータ (インターネット上びコンピュータを含む)\] の既定値を設定することにより、この脆弱性のスコープを削減することができます。 これは、インターネットからの攻撃の可能性の軽減に役立ちます。

    **注** **:** Windows Small Business Server 2003 はリモート Web ワークプレースという名前の機能を使用します。 この機能は TCP ポート 4125 を使用して RDP の接続をリッスンします。 この機能を使用している場合、このポートが、ポート 3389 だけでなくインターネットからブロックされていることを確認しなければなりません。

    **注:** ほかのポートを使用するために、影響のあるコンポーネントを手動で変更することが可能です。 これを行なった場合、これらの追加のポートもブロックしてください。

-   **IPSec** **ポリシーを使用して、リモートデスクトップ接続をセキュアにする手助けをします。**

    特定の設定は個々の環境によります。 インターネット プロトコル セキュリティ (IPsec) に関する情報は、次の [Web サイト](http://www.microsoft.com/windowsserver2003/technologies/networking/ipsec/default.mspx) (英語情報) をご覧ください。IPsec およびフィルタの適用方法に関する詳細は、サポート技術情報 [313190](http://support.microsoft.com/kb/313190) および [813878](http://support.microsoft.com/kb/813878) をご覧ください。

-   **仮想プライベートネットワーク** **(VPN)** **接続を使用して、リモートデスクトップ接続をセキュアにする手助けをします。**

    特定の設定は個々の環境によります。 仮想プライベート ネットワークに関する詳細情報は、次の [Web サイト](http://technet.microsoft.com/en-us/network/bb545442.aspx) (英語情報) をご覧ください。

#### 「リモート デスクトップ プロトコルの脆弱性」のよく寄せられる質問 - CAN-2005-1218:

**どのようなことが起こる可能性がありますか?**

これは、[サービス拒否](http://www.microsoft.com/japan/security/glossary.mspx) の脆弱性です。攻撃者はこの脆弱性を悪用し、影響を受けるコンピュータが応答しないようにし、また自動的に再起動させる可能性があります。その間、サーバーはリクエストに応答できません。攻撃者により、サービス拒否の脆弱性が悪用されても、コードが実行されたり、特権の昇格が行われることはないと考えらます。しかし、影響を受けるコンピュータがリクエストを受け付けなくなる可能性があります。

**何が原因で起こりますか?**

リモート デスクトップ プロトコールによるデータの確認に使用される処理方法が原因です。

**リモートデスクトッププロトコル** **(RDP)とは何ですか？**

リモート デスクトップ プロトコル (RDP) を利用すると、ユーザーはデスクトップのコンピュータに仮想セッションを作成することができます。 これにより、リモートのユーザーがコンピュータ上の全てのデータとアプリケーションにアクセス可能になります。 RDP に関する詳細情報は、次の [Web サイト](http://msdn2.microsoft.com/en-us/library/aa383015.aspx) (英語情報) をご覧ください。

**マイクロソフトのどの製品に** **RDP** **が搭載されているのですか?**

通常、RDP は Windows の機能の基本的なプロトコルであり、リモートでのデスクトップのセッションを可能にします。 たとえば、 これらの機能には、以下が含まれます。Windows 2000 および Windows Server 2003 のターミナル サービスは RDP を実装しています。 ターミナルサービスおよび RDP に関する詳細情報は、次の[Web サイト](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/technologies/featured/termserv/default.mspx) (英語情報) をご覧ください。

Windows XP の [リモート デスクトップ](http://www.microsoft.com/windowsxp/downloads/tools/rdclientdl.mspx) (英語情報) は RDP を実装します。 Windows XP のリモート デスクトップの機能に関する詳細は、次の [Web サイト](http://www.microsoft.com/japan/windowsxp/using/mobility/getstarted/default.mspx) をご覧ください。

Windows XP および Windows Server 2003 でリモート アシスタンスのリクエストを作成すると、リクエストの期限が切れた後も短期間 RDP が有効になります。 手動で、およびグループ ポリシーを使用してリモート アシスタンスを無効にする方法など、リモート アシスタンスに関する情報は [Web サイト](http://www.microsoft.com/technet/prodtechnol/winxppro/maintain/intmgmt/20_xprem.mspx) (英語情報) をご覧ください。

Windows XP Media Center Edition 2005 のコンピュータの Media Center Extenders は RDP を有効にします。 Media Center Extenders に関する詳細情報は、次の [Web サイト](http://msdn2.microsoft.com/en-us/library/ms123402.aspx) をご覧ください。 Media Center Extenders に関する技術的な詳細情報は、次の[Web サイト](http://msdn2.microsoft.com/en-us/library/ms816292.aspx) (英語情報) をご覧ください。 Media Center Extenders は既定の TCP 3389 ポート以上のポートで、RDP を使用できます。

Windows Small Business Server 2003 のリモート Web ワークプレースは RDP を有効にします。 この機能に関する詳細については、[TechNet Support WebCast](http://support.microsoft.com/kb/833983/en-us) (英語情報) をご覧ください。 Windows Small Business Server 2003 のリモート Web ワークプレースの機能を無効にする方法に関する詳細は、次の [Web サイト](http://www.microsoft.com/technet/security/secnews/articles/sec_sbs2003_network.mspx) (英語情報) をご覧ください。 リモート Web ワークプレースの機能は、既定の TCP 3389 ポート以上のポートで、RDP を使用します。

**Windows** **で、RDP** **は既定で有効になるのですか?**

いいえ。リモート デスクトップ プロトコル (RDP) は、どのオペレーティング システムのバージョンでも、既定で有効ではありません。 Windows XP Media Center Edition のいくつかのバージョンでは、Media Center Extender がインストールされている場合に RDP が有効です。Media Center Extenders に関する詳細情報は、次の [Web サイト](http://msdn2.microsoft.com/en-us/library/ms123402.aspx) (英語情報) をご覧ください。

Media Center Extenders に関する技術的な詳細情報は、次の[Web サイト](http://msdn2.microsoft.com/en-us/library/ms816292.aspx) (英語情報) をご覧ください。

Small Business Server 2000 および Windows Small Business Server 2003 では、RDP は既定で有効にされています。 しかし、Windows Small Business Server 2003 およびそれ以前のバージョンでは、既定で RDP のプロトコルの通信ポートがインターネットからブロックされています。そして RDP は、ターミナル サービスまたはリモート Web ワークプレースが電子メールとインターネット接続の構成ウィザード (CEICW) を使用して有効にされていない限り、ローカル ネットワークでのみ利用可能です。

Windows XP Home Edition はリモート デスクトップをサポートしていません。 しかし、リモート アシスタンスをサポートしています。 リモート アシスタンスは、リモート アシスタンスのリクエストの期限が切れた後も短期間 RDP を有効にします。 この間、Windows XP Home のコンピュータが Windows ファイアウォールによるリモート デスクトップの例外を許可した場合に、この問題に対する脆弱性の影響を受ける可能性があります。

**この脆弱性により、攻撃者は何を行う可能性がありますか?**

攻撃者はこの脆弱性を悪用し、影響を受けるコンピュータが応答しないようにする可能性があります。

**どのような人物によりこの脆弱性が悪用される可能性がありますか?**

この脆弱性の影響を受けるコンピュータに、特別な細工をしたメッセージを送信できる匿名ユーザーが、この脆弱性を悪用する可能性があります。

**どのように攻撃者はこの脆弱性を悪用する可能性がありますか?**

攻撃者は、特別な細工をしたメッセージを作成し、そのメッセージを影響を受けるコンピュータに送信することにより、この脆弱性を悪用する可能性があります。このようなメッセージを受け取ると、この脆弱性の影響を受けるコンピュータは異常終了し、サービス拒否が起こる可能性があります。

**主にどのようなコンピュータがこの脆弱性による危険にさらされますか?**

Windows 2000 Server ベースのターミナル サーバーおよび Windows Server 2003 ベースのターミナル サーバーが主にこの脆弱性による危険にさらされます。 管理者が、ターミナル サーバー機能を有効にするためにこのオペレーティング システムを手動で構成した場合に、この問題の影響を受けます。 Windows Small Business Server 2003 ベースのサーバーも、管理者がインターネットからサーバーへのターミナル サービスおよびリモート Web ワークプレースの接続を有効にするため、電子メールとインターネット接続の構成ウィザードを使用した場合に影響を受ける可能性があります。 Windows XP および Windows Server 2003 のコンピュータは手動でリモート デスクトップが有効にされている場合、またはリモート アシスタンスを使用している場合に危険にさらされます。 Windows 2000 Professional は RDP のプロトコルのサポートを含んでおらず、この問題の影響は受けません。

**この脆弱性がインターネットで悪用される可能性はありますか?**

はい。攻撃者はこの脆弱性をインターネットで悪用する可能性があります。ファイアウォールによる最善策および標準の既定の構成により、インターネット経由での攻撃からコンピュータを保護することができます。マイクロソフトはどのようにコンピュータを保護するかについての情報を提供しています。ホーム ユーザーの方は、[Protect Your PC Web サイト](http://www.microsoft.com/japan/athome/security/protect/default.aspx) をご覧ください。IT プロフェッショナルは、[セキュリティ ガイダンス センター](http://www.microsoft.com/japan/security/guidance/default.mspx) をご覧ください。

**この更新プログラムは何を修正しますか?**

この累積的な更新プログラムは、RDP がメッセージを読み込む間にメッセージの長さを検証する方法を変更することにより、この脆弱性を排除します。

**このセキュリティ情報のリリース時に、この脆弱性は一般に公開されていたのですか?**

はい。この脆弱性は一般に公開されていました。

**このセキュリティ情報のリリース時に、マイクロソフトはこの脆弱性が悪用されたという報告を受けていましたか?**

いいえ。このセキュリティ情報が最初にリリースされた段階で、マイクロソフトはこの脆弱性が悪用され、お客様が攻撃されたということを示す情報は受けておらず、また、公開された検証用コードのいかなる実例の存在も確認しておりません。

**このセキュリティ更新プログラムを適用すると、公開されている脆弱性からコンピュータを保護する手助けとなりますか?**

はい。 このセキュリティ更新プログラムはこのセキュリティ情報で公開された脆弱性に対処するものです。

    1.  **\[スタート\]** をクリックし、次に **\[検索\]** をクリックします。

    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に **\[プロパティ\]** をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

    5.  **\[バージョン情報\]** タブで、適切なファイル情報の表に記載されているバージョンと比較し、コンピュータにインストールされているファイルのバージョンを確認します。

-   **Microsoft Baseline Security Analyzer**

    影響を受けるコンピュータにセキュリティ更新プログラムがインストールされていることを確認するためには、Microsoft Baseline Security Analyzer (MBSA) ツールを使用してください。管理者は、一般的なセキュリティ上誤った構成とともに、不足しているセキュリティ更新プログラムについて、ローカルコンピュータおよびリモート コンピュータを Microsoft Baseline Security Analyzer (MBSA) を使ってスキャンすることができます。MBSA に関する追加情報は、[Microsoft Baseline Security Analyzer](http://technet.microsoft.com/ja-jp/security/cc184924.aspx) の Web サイトをご覧ください。

-   **ファイルバージョンの確認**

    **注:** Microsoft Windows にはいくつかのバージョンがあるため、次のステップは使用中のコンピュータにより異なる場合があります。その場合、製品の説明書をご覧ください。

    1.  **\[スタート\]** をクリックし、次に **\[検索\]** をクリックします。
    2.  **\[検索結果\]** のウィンドウの **\[検索コンパニオン\]** の下の **\[ファイルとフォルダすべて\]** をクリックします。

    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に **\[プロパティ\]** をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

**更新プログラムが適用されたかどうかを確認する方法**

-   **Microsoft Baseline Security Analyzer**

    影響を受けるコンピュータにセキュリティ更新プログラムがインストールされていることを確認するためには、Microsoft Baseline Security Analyzer (MBSA) ツールを使用してください。管理者は、一般的なセキュリティ上誤った構成とともに、不足しているセキュリティ更新プログラムについて、ローカルコンピュータおよびリモート コンピュータを Microsoft Baseline Security Analyzer (MBSA) を使ってスキャンすることができます。MBSA に関する追加情報は、[Microsoft Baseline Security Analyzer](http://technet.microsoft.com/ja-jp/security/cc184924.aspx) の Web サイトをご覧ください。

-   **ファイルバージョンの確認**

    **注:** Microsoft Windows にはいくつかのバージョンがあるため、次のステップは使用中のコンピュータにより異なる場合があります。その場合、製品の説明書をご覧ください。

    1.  **\[スタート\]** をクリックし、次に **\[検索\]** をクリックします。
    2.  **\[検索結果\]** のウィンドウの **\[検索コンパニオン\]** の下の **\[ファイルとフォルダすべて\]** をクリックします。
    3.  **\[ファイル名のすべてまたは一部\]** のボックスで、適切なファイル情報の表からファイル名を入力し、次に **\[検索\]** をクリックします。
    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に **\[プロパティ\]** をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

    5.  **\[バージョン情報\]** タブで、適切なファイル情報の表に記載されているバージョンと比較し、コンピュータにインストールされているファイルのバージョンを確認します。

        **注:** ファイルのバージョン以外の属性はインストール中に変更される場合があります。そのほかのファイルの属性をファイル情報の表の情報と比較することは、更新プログラムが正しくインストールされたことを確認する方法としてサポートされていません。また、ファイル名がインストール中に変更される場合があります。ファイルまたはバージョンの情報が存在しない場合、その他の方法によって更新プログラムが正しくインストールされたことを確認してください。

-   **レジストリキーの確認**

    また、次のレジストリ キーを調べることにより、このセキュリティ更新プログラムがインストールしたファイルを確認することもできます。

    HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Updates\\Windows 2000\\SP5\\KB899591\\Filelist

    **注:** このレジストリ キーには、インストールされるファイルの完全な一覧が含まれない場合があります。また、管理者または OEM メーカーによって899591 のセキュリティ更新プログラムを Windows インストール ソース ファイルに統合またはスリップストリーム化されている場合、このレジストリ キーが適切に作成されない場合があります。
