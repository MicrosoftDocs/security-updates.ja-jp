Security Bulletin

マイクロソフト セキュリティ情報 MS09-020 - 緊急
===============================================

インターネット インフォメーション サービス (IIS) の脆弱性により、特権が昇格される (970483)
------------------------------------------------------------------------------------------

公開日: 2009年6月10日 | 最終更新日: 2009年6月18日

**バージョン:** 1.0

[](http://www.microsoft.com/japan/security/bulletins/ms09-020e.mspx)[![](images/Dn627102.SNS300x50(ja-JP,Security.10).jpg)](https://profile.microsoft.com/regsysprofilecenter/subscriptionwizard.aspx?wizid=cbdde499-aa75-4a75-9cb3-f48eac2e7c3f&lcid=1041)

### 概説

#### 要点

このセキュリティ更新プログラムは Microsoft インターネット インフォメーション サービス (IIS) に存在する 1 件の一般に公開された脆弱性および 1 件の非公開で報告された脆弱性を解決します。これらの脆弱性で、攻撃者が特別な細工がされた HTTP リクエストを、認証を必要とする Web サイトに送信した場合、特権が昇格される可能性があります。これらの脆弱性により、攻撃者により、許可する認証の種類が指定されている IIS の構成が回避される可能性がありますが、特定のユーザーによりアクセスできるファイルを検証するファイル システム ベースのアクセス制御リスト (ACL) のチェックが回避されることはありません。これらの脆弱性が悪用された場合でも、攻撃者の行動はファイル システムの ACL により匿名ユーザー アカウントへ与えられた許可の範囲に制限されます。

このセキュリティ更新プログラムは、Microsoft Windows 2000、Windows XP および Windows Server 2003 のすべてのサポートされているエディション上の Microsoft インターネット インフォメーション サービスについて、深刻度は「重要」と評価されています。

詳細情報は、このセクションの「影響を受けるソフトウェアおよび影響を受けないソフトウェア」のサブセクションをご覧ください。 このセキュリティ更新プログラムは IIS の WebDAV 拡張が HTTP リクエストを処理する方法を変更することにより、これらの脆弱性を解決します。この脆弱性に関する詳細情報は、次の「脆弱性の情報」のセクションの下の特定の脆弱性のエントリに関するサブセクション「よく寄せられる質問 (FAQ)」をご覧ください。

このセキュリティ更新プログラムは、[マイクロソフト セキュリティ アドバイザリ 971492](http://technet.microsoft.com/security/advisory/971492) で説明した脆弱性を解決します。

**推奨する対応策:** 自動更新を有効にしている場合、このセキュリティ更新プログラムが自動的にダウンロードされインストールされるため、特別な操作を行う必要はありません。自動更新を有効にしていない場合、この更新プログラムを手動で確認し、インストールする必要があります。自動更新の具体的な構成オプションに関する情報は、サポート技術情報 [294781](http://support.microsoft.com/kb/294871) をご覧ください。

管理者およびエンタープライズのインストール、またはこのセキュリティ更新プログラムを手動でインストールしたいエンドユーザーの場合は、更新プログラムの管理ソフトウェアまたは [Microsoft Update サービス](http://update.microsoft.com/microsoftupdate)で更新プログラムを確認して、この更新プログラムをできる限り早期に適用することを推奨します。このセキュリティ情報の後半の「検出および適用のツールとガイダンス」をご覧ください。

IIS 5.0 の WebDAV の認証回避の脆弱性 - CVE-2009-1122  
----------------------------------------------------
  
<span></span>
IIS の WebDAV 拡張が HTTP リクエストを処理する方法に特権の昇格の脆弱性が存在します。攻撃者は、特別な細工がされた匿名の HTTP リクエストを作成し、通常は認証が必要な場所へアクセスできる可能性があります。
  
Common Vulnerabilities and Exposures のリストの標準のエントリとしてこの脆弱性をご覧になるためには、[CVE-2009-1122](http://cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1122) をご覧ください。
  
#### 「IIS 5.0 の WebDAV の認証回避の脆弱性」の問題を緩和する要素 - CVE-2009-1122:
  
「問題を緩和する要素」とは、設定、一般的な構成または一般的な最善策、既定の状態により、脆弱性の悪用の深刻度が低くなる可能性がある要素を指します。次の「問題を緩和する要素」は、お客様の環境で役立つ場合があります。
  
-   Microsoft Internet Information Services 5.1、6.0 および 7.0 はこの脆弱性の影響を受けません。  
-   **既定で、匿名ユーザー アカウントの書き込みアクセス権が拒否されます。**この脆弱性が書き込みアクセス権を伴って悪用されるには、匿名ユーザー アカウントが IIS フォルダー構造に ACL の書き込みアクセス権が設定されていることが必要になります。しかし、既定で匿名ユーザー アカウントに設定されているのは、読み取りアクセス権のみです。IIS 6.0 では、アクセス制御エントリ (ACE) は明示的に既定の匿名ユーザーアカウントを拒否します。管理者が無効にしない限り、既定の Web サイトのルートの下のすべての子がアクセス拒否 ACE を継承します。  
-   **ファイル システムの ACL は有効です。**本脆弱性により、どの認証を許可するのかを指定した IIS の構成を回避可能ですが、ファイル システムの ACL を確認し、指定されたユーザーがファイルにアクセス可能かどうかの検証に関しては回避できません。この脆弱性が悪用された場合でも、攻撃者の行動はファイル システムの ACL レベルで匿名ユーザー アカウントへ与えられた許可の範囲に制限されます。従って、この脆弱性はファイル システムの ACL で匿名ユーザー アカウントに与えられたアクセス許可を超えて悪用されることはありません。既定で、匿名ユーザー アカウントの構成は IUSR\_&lt;コンピューター名&gt; アカウントです。
  
#### 「IIS 5.0 の WebDAV の認証回避の脆弱性」の回避策 - CVE-2009-1122:
  
回避策は、設定または構成の変更を示しており、基本的な脆弱性を正すものではありませんが、更新プログラムを適用する前に既知の攻撃方法を阻止するのに役立ちます。マイクロソフトは、以下の回避策を検証し、この回避策が機能性を低下させるかどうかについて、説明の部分で述べています。
  
-   **WebDAV を無効にする**
  
    サーバーで WebDAV の機能を必要としない場合、無効にすることができます。 IIS 5.0 では、サポート技術情報 [241520](http://support.microsoft.com/kb/241520) で説明している方法で WebDAV を無効にすることができます。
  
    **回避策の影響** **:** この回避策の適用後、IIS は WebDAV のリクエストを処理しなくなります。
  
-   **IIS 5.0 で WebDAV を無効にする別の方法**
  
    [IIS Lockdown Tool 2.1](http://www.microsoft.com/downloads/details.aspx?familyid=dde9efc0-bb30-47eb-9a61-fd755d23cdec&displaylang=en) を使用して WebDAV を無効にすることができます。マイクロソフト サポート技術情報 [325864](http://support.microsoft.com/kb/325864) をご覧ください。
  
    -   今すぐインストールを開始するには、\[開く\] または \[このプログラムを現在の場所から実行する\] をクリックします。  
    -   後日インストールするために、このダウンロードをコンピューターにコピーするには \[保存\] または \[このプログラムをディスクに保存する\] をクリックします。
  
    **回避策の影響** **:** この方法は [URLScan セキュリティ ツール](http://technet.microsoft.com/ja-jp/security/cc242650.aspx)をインストールすることにより、行うことができます。既定で、URLScan は WebDAV にマップされる可能性のある HTTP 動詞またはヘッダーを検出することにより、WebDAV へのリクエストを禁止します IIS Lockdown ツールをアンインストールしても、URLScan は残ります。URLScan をアンインストールするには、URLScan ISAPI フィルターを IIS マネージャーのスナップインから登録を解除します。
  
-   **ファイル システムの ACL を変更し、匿名ユーザーのアカウントへのアクセスを拒否する**
  
    WebDAV の機能が必要な場合、サーバー上のリソースへのアクセスを制御するために NTFS ACL を使用する必要があります。WebDAV 経由で匿名ユーザーにリソースをアクセスさせない場合、そのリソースへの必要なアクセスに基づき、適切な読み取りまたは書き込み NTFS ACL を適用する必要があります。IIS コンテンツに ACL を設定することに関する詳細情報は、次のサポート技術情報をご覧ください。
  
    -   マイクロソフト サポート技術情報 [271071](http://support.microsoft.com/kb/271071)  
    -   マイクロソフト サポート技術情報 [812614](http://support.microsoft.com/kb/812614)
  
    **回避策の影響** **:** この回避策を適用すると、IUSR\_&lt;コンピューター名&gt; アカウントによりコンテンツをアクセスさせる必要がある製品では悪影響が起きる可能性があります。このような製品の例には、System Center Configuration Manager 2007 があります。
  
#### 「IIS 5.0 の WebDAV の認証回避の脆弱性」のよく寄せられる質問 - CVE-2009-1122:
  
**どのようなことが起こる可能性がありますか?**  
これは特権の昇格の脆弱性で、攻撃者により IIS Server 上のコンテンツにアクセスするための認証要件が回避される可能性があります。
  
**何が原因で起こりますか?**  
WebDAV 拡張が特別な細工がされた URL リクエストを適切にデコードしないため、この脆弱性が起こります。このため、リクエストを処理する際に WebDAV が誤った構成を適用します。適用された構成が匿名アクセスを許可している場合、特別な細工がされたリクエストが認証を回避します。さらに、WebDAV が間違った構成を適用するため、IP アドレスの制限など、特定のパスに適用されるべき IIS の構成のその他の設定が WebDAV により受け入れられない可能性があります。
  
なお、IIS は、匿名ユーザー アカウントに設定されているアクセス権の範囲でリクエストを処理する事に留意してください。このため、NTFS ACL を回避する目的でこの脆弱性が悪用されることはありません。ファイル システムの ACL による匿名ユーザー アカウントの制限は継承されます。
  
**匿名ユーザー アカウントとは何ですか?**  
Web サーバーで実行している Web サイトおよびアプリケーションに匿名のアクセス許可が必要な場合、匿名アクセス用の特別なユーザー アカウントを用いて IIS を構成する必要があります。その後、匿名の Web ユーザーがリクエストした際、ファイル システムのコンテンツにアクセスする時に、このアカウントが使用されます。既定の構成では、IIS は匿名ユーザー アカウントに IUSR\_&lt;コンピューター名&gt; を使用します。&lt;コンピューター名&gt; は IIS を実行しているコンピューター名です。しかし、匿名ユーザー アカウントとして別のローカル ユーザー アカウントまたはドメインのユーザー アカウントを使用して、匿名ユーザーの ID を変更することが可能です。
  
**WebDAV とは何ですか?**  
Web Distributed Authoring and Versioning (WebDAV) とはHTTP (Hypertext Transfer Protocol) の拡張仕様で、ファイルのコピー、移動、削除および作成などの基本的な機能をコンピューターが実行する方法をHTTP を使用して定義します。
  
**アクセス制御リスト (ACL) とは何ですか?**  
アクセス制御リスト (ACL) とは、オブジェクトに適用されるセキュリティ保護のリストです。(ファイル、プロセス、イベントまたはセキュリティ記述子を持つものがオブジェクトです。) ACL のエントリはアクセス制御エントリ (ACE) です。アクセス制御リストには随意とシステムの 2 種類があります。ACL の各 ACE はそのトラスティに許可された、拒否された、または監査されたアクセス権限を識別します。詳細情報は、MSDN コラム [Access Control Lists](http://msdn.microsoft.com/library/aa374872(vs.85).aspx) (英語情報) をご覧ください。
  
**この脆弱性により、攻撃者は何を行う可能性がありますか?**  
この脆弱性により、攻撃者により、許可する認証の種類が指定されている IIS の構成が回避される可能性がありますが、特定のユーザーによりアクセスできるファイルを検証するファイル システム ベースのアクセス制御リスト (ACL) のチェックが回避されることはないと考えられます。この脆弱性が悪用された場合でも、攻撃者の行動はファイル システムの ACL レベルで匿名ユーザー アカウントへ与えられたアクセス許可の範囲に制限されます。攻撃者は認証されたユーザーにのみアクセス可能であるはずの情報に匿名でアクセスしたり、匿名ユーザー アカウントが書き込みアクセス許可を与えられていない場所にファイルを書き込む可能性があります。
  
**攻撃者はこの脆弱性をどのように悪用する可能性がありますか?**  
攻撃者がこの脆弱性を悪用した場合、認証が必要な Web サイトに対して特別に細工した HTTP リクエストを作成し、保護されたリソースに対し不正アクセスを行う可能性があります。
  
**主にどのようなコンピューターがこの脆弱性による危険にさらされますか?**  
IIS バージョン 5.0 が使用されており、WebDAV が有効にされている Microsoft Windows 2000 システムが主にこの脆弱性の影響を受けます。通常、このようなシステムは Web サーバーやアプリケーション サーバーですが、開発者や Web デザイナー、IIS が WebDAV とともに使用されているワークステーション システムも危険にさらされる可能性があります。匿名ユーザー アカウントが書き込みアクセス許可を与えられている既定の構成から外れているシステムは、危険にさらされる可能性がさらに高くなります。
  
**この更新プログラムはどのように問題を修正しますか?**  
このセキュリティ更新プログラムは IIS の WebDAV 拡張が HTTP リクエストを処理する方法を変更することにより、この脆弱性を解決します。

いいえ。マイクロソフトは信頼のおける情報元からこの脆弱性に関する情報を受けました。マイクロソフトは、このセキュリティ情報が最初に公開された際に、この脆弱性が一般に知られていたという情報は受けていませんでした。
  
**このセキュリティ情報の公開時に、マイクロソフトはこの脆弱性が悪用されたという報告を受けていましたか?**  
いいえ。このセキュリティ情報が最初に公開された段階で、マイクロソフトはこの脆弱性が悪用され、お客様が攻撃されたということを示す情報は受けておらず、また、公開された検証用コードのいかなる実例の存在も確認しておりません。
  
IIS 5.1 および 6.0 の WebDAV の認証回避の脆弱性 - CVE-2009-1535  
---------------------------------------------------------------
  
<span></span>
IIS の WebDAV 拡張が HTTP のリクエストを処理する方法に特権の昇格の脆弱性が存在します。攻撃者がこの脆弱性を悪用した場合、特別に細工した匿名の HTTP リクエストを作成し、通常は認証が必要な場所へアクセスできる可能性があります。
  
Common Vulnerabilities and Exposures のリストの標準のエントリとしてこの脆弱性をご覧になるためには、[CVE-2009-1535](http://cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-1535) をご覧ください。
  
#### 「IIS 5.1 および 6.0 の WebDAV の認証回避の脆弱性」の問題を緩和する要素 - CVE-2009-1535:
  
「問題を緩和する要素」とは、設定、一般的な構成または一般的な最善策、既定の状態により、脆弱性の悪用の深刻度が低くなる可能性がある要素を指します。次の「問題を緩和する要素」は、お客様の環境で役立つ場合があります。
  
-   Microsoft Internet Information Services version 5.0 および 7.0 はこの脆弱性の影響を受けません。  
-   **IIS 6.0 で、WebDAV は既定で無効にされています。**IIS 6.0 を実行している Windows Server 2003 コンピューターでは、既定の構成で WebDAV が無効にされています。これらのコンピューターでは、管理者が WebDAV を有効にしない限り、この脆弱性が危険にさらされることはありません。  
-   **既定で、匿名ユーザー アカウントの書き込みアクセス権が拒否されます。**この脆弱性が書き込みアクセス権を伴って悪用されるには、匿名ユーザー アカウントが IIS フォルダー構造に ACL の書き込みアクセス権が設定されていることが必要になります。しかし、既定で匿名ユーザー アカウントに設定されているのは、読み取りアクセス権のみです。IIS 6.0 では、アクセス制御エントリ (ACE) は明示的に既定の匿名ユーザーアカウントを拒否します。管理者が無効にしない限り、既定の Web サイトのルートの下のすべての子がアクセス拒否 ACE を継承します。  
-   **ファイル システムの ACL は有効です。**本脆弱性により、どの認証を許可するのかを指定した IIS の構成を回避可能ですが、ファイル システムの ACL を確認し、指定されたユーザーがファイルにアクセス可能かどうかの検証に関しては回避できません。この脆弱性が悪用された場合でも、攻撃者の行動はファイル システムの ACL レベルで匿名ユーザー アカウントへ与えられた許可の範囲に制限されます。従って、この脆弱性はファイル システムの ACL で匿名ユーザー アカウントに与えられたアクセス許可を超えて悪用されることはありません。既定で、匿名ユーザー アカウントの構成は IUSR\_&lt;コンピューター名&gt; アカウントです。
  
#### 「IIS 5.1 および 6.0 の WebDAV の認証回避の脆弱性」の回避策 - CVE-2009-1535:
  
回避策は、設定または構成の変更を示しており、基本的な脆弱性を正すものではありませんが、更新プログラムを適用する前に既知の攻撃方法を阻止するのに役立ちます。マイクロソフトは、以下の回避策を検証し、この回避策が機能性を低下させるかどうかについて、説明の部分で述べています。
  
-   **WebDAV を無効にする**
  
    サーバーで WebDAV の機能を必要としない場合、無効にすることができます。
  
    IIS 5.0 および IIS 5.1 では、サポート技術情報 [241520](http://support.microsoft.com/kb/241520) で説明している方法で WebDAV を無効にすることができます。
  
    IIS 6.0 では、次のステップにより WebDAV を無効にすることができます。
  
    1.  IIS マネージャーの MMC スナップインを起動します。  
    2.  左ウィンドウの、ローカル コンピューターのアイコンを展開します。  
    3.  この項目の下の \[Web サービス拡張\] をクリックします。  
    4.  右ウィンドウで、\[WebDAV\] を選択し、\[禁止\] ボタンをクリックします。
  
    **回避策の影響** **:** この回避策の適用後、IIS は WebDAV のリクエストを処理しなくなります。
  
-   **IIS 5.1 および IIS 6.0 上の WebDAV を無効にする別の方法**
  
    Microsoft URLScan Filter v3.1 を使用して WebDAV を無効にすることもできます。
  
    IIS 5.1 および IIS 6.0 について、Microsoft URLScan Filter v3.1 を次の Web サイトからダウンロードしてください。
  
    -   32ビット システム用: [Microsoft Urlscan Filter v3.1 (x86)](http://www.microsoft.com/downloads/details.aspx?familyid=ee41818f-3363-4e24-9940-321603531989&displaylang=en)  
    -   x64-based システム用 [Microsoft Urlscan Filter v3.1 (x64)](http://www.microsoft.com/downloads/details.aspx?familyid=361e5598-c1bd-46b8-b3e7-3980e8bdf0de&displaylang=en)
  
    次のいずれかを実行します:
  
    -   今すぐインストールを開始するには、\[開く\] または \[このプログラムを現在の場所から実行する\] をクリックします。  
    -   後日インストールするために、このダウンロードをコンピューターにコピーするには \[保存\] または \[このプログラムをディスクに保存する\] をクリックします。
  
    **回避策の影響** **:** この方法は [URLScan セキュリティ ツール](http://technet.microsoft.com/ja-jp/security/cc242650.aspx)をインストールすることにより、行うことができます。既定で、URLScan は WebDAV にマップされる可能性のある HTTP 動詞またはヘッダーを検出することにより、WebDAV へのリクエストを禁止します
  
-   **ファイル システムの ACL を変更し、匿名ユーザーのアカウントへのアクセスを拒否する**
  
    WebDAV の機能が必要な場合、サーバー上のリソースへのアクセスを制御するために NTFS ACL を使用する必要があります。WebDAV 経由で匿名ユーザーにリソースをアクセスさせない場合、そのリソースへの必要なアクセスに基づき、適切な読み取りまたは書き込み NTFS ACL を適用する必要があります。IIS コンテンツに ACL を設定することに関する詳細情報は、次のサポート技術情報をご覧ください。
  
    -   マイクロソフト サポート技術情報 [271071](http://support.microsoft.com/kb/271071)  
    -   マイクロソフト サポート技術情報 [812614](http://support.microsoft.com/kb/812614)
  
    **回避策の影響** **:** この回避策を適用すると、IUSR\_&lt;コンピューター名&gt; アカウントによりコンテンツをアクセスさせる必要がある製品では悪影響が起きる可能性があります。このような製品の例には、System Center Configuration Manager 2007 があります。

**更新プログラムが適用されたかどうかを確認する方法**

-   **Microsoft Baseline Security Analyzer**

    影響を受けるコンピューターにセキュリティ更新プログラムがインストールされていることを確認するためには、Microsoft Baseline Security Analyzer (MBSA) ツールを使用してください。詳細については、このセキュリティ情報の上部にある「検出および展開ツールとガイダンス」のセクションをご覧ください。

-   **ファイルバージョンの確認**

    Microsoft Windows にはいくつかのバージョンがあるため、次のステップは使用中のコンピューターにより異なる場合があります。その場合、製品の説明書をご覧ください。

    1.  \[スタート\] をクリックし、次に \[検索\] をクリックします。
    2.  \[検索結果\] のウィンドウの \[検索コンパニオン\] の下の \[ファイルとフォルダすべて\] をクリックします。
    3.  \[ファイル名のすべてまたは一部\] のボックスで、適切なファイル情報の表からファイル名を入力し、次に \[検索\] をクリックします。
    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に \[プロパティ\] をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

    5.  \[バージョン情報\] タブで、適切なファイル情報の表に記載されているバージョンと比較し、コンピューターにインストールされているファイルのバージョンを確認します。

        **注:** ファイルのバージョン以外の属性はインストール中に変更される場合があります。そのほかのファイルの属性をファイル情報の表の情報と比較することは、更新プログラムが正しくインストールされたことを確認する方法としてサポートされていません。また、ファイル名がインストール中に変更される場合があります。ファイルまたはバージョンの情報が存在しない場合、その他の方法によって更新プログラムが正しくインストールされたことを確認してください。

-   **レジストリキーの確認**

    このセクションの「参照表」にリストされているレジストリ キーを調べることにより、セキュリティ更新プログラムがインストールしたファイルを確認することもできます。

    これらのレジストリ キーには、インストールされるファイルの完全な一覧が含まれない場合があります。また、管理者もしくは OEM メーカーによって、セキュリティ更新プログラムを Windows インストール ソース ファイルに統合またはスリップストリーム化されている場合、レジストリ キーが適切に作成されない場合があります。

**更新プログラムが適用されたかどうかを確認する方法**

-   **Microsoft Baseline Security Analyzer**

    影響を受けるコンピューターにセキュリティ更新プログラムがインストールされていることを確認するためには、Microsoft Baseline Security Analyzer (MBSA) ツールを使用してください。詳細については、このセキュリティ情報の上部にある「検出および展開ツールとガイダンス」のセクションをご覧ください。

-   **ファイルバージョンの確認**

    Microsoft Windows にはいくつかのバージョンがあるため、次のステップは使用中のコンピューターにより異なる場合があります。その場合、製品の説明書をご覧ください。

    1.  \[スタート\] をクリックし、次に \[検索\] をクリックします。
    2.  \[検索結果\] のウィンドウの \[検索コンパニオン\] の下の \[ファイルとフォルダすべて\] をクリックします。
    3.  \[ファイル名のすべてまたは一部\] のボックスで、適切なファイル情報の表からファイル名を入力し、次に \[検索\] をクリックします。
    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に \[プロパティ\] をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

    5.  \[バージョン情報\] タブで、適切なファイル情報の表に記載されているバージョンと比較し、コンピューターにインストールされているファイルのバージョンを確認します。

        **注:** ファイルのバージョン以外の属性はインストール中に変更される場合があります。そのほかのファイルの属性をファイル情報の表の情報と比較することは、更新プログラムが正しくインストールされたことを確認する方法としてサポートされていません。また、ファイル名がインストール中に変更される場合があります。ファイルまたはバージョンの情報が存在しない場合、その他の方法によって更新プログラムが正しくインストールされたことを確認してください。

-   **レジストリキーの確認**

    このセクションの「参照表」にリストされているレジストリ キーを調べることにより、セキュリティ更新プログラムがインストールしたファイルを確認することもできます。

    これらのレジストリ キーには、インストールされるファイルの完全な一覧が含まれない場合があります。また、管理者もしくは OEM メーカーによってセキュリティ更新プログラムを Windows インストール ソース ファイルに統合またはスリップストリーム化されている場合、レジストリ キーが適切に作成されない場合があります。

**更新プログラムが適用されたかどうかを確認する方法**

-   **Microsoft Baseline Security Analyzer**

    影響を受けるコンピューターにセキュリティ更新プログラムがインストールされていることを確認するためには、Microsoft Baseline Security Analyzer (MBSA) ツールを使用してください。詳細については、このセキュリティ情報の上部にある「検出および展開ツールとガイダンス」のセクションをご覧ください。

-   **ファイルバージョンの確認**

    Microsoft Windows にはいくつかのバージョンがあるため、次のステップは使用中のコンピューターにより異なる場合があります。その場合、製品の説明書をご覧ください。

    1.  \[スタート\] をクリックし、次に \[検索\] をクリックします。
    2.  \[検索結果\] のウィンドウの \[検索コンパニオン\] の下の \[ファイルとフォルダすべて\] をクリックします。
    3.  \[ファイル名のすべてまたは一部\] のボックスで、適切なファイル情報の表からファイル名を入力し、次に \[検索\] をクリックします。
    4.  ファイルの一覧で、適切なファイル情報の表からファイル名を右クリックし、次に \[プロパティ\] をクリックします。

        **注:** インストールされているオペレーティングシステムまたはプログラムのバージョンにより、ファイル情報の表に記載されているファイルで、インストールされないものがある場合もあります。

    5.  \[バージョン情報\] タブで、適切なファイル情報の表に記載されているバージョンと比較し、コンピューターにインストールされているファイルのバージョンを確認します。

        **注:** ファイルのバージョン以外の属性はインストール中に変更される場合があります。そのほかのファイルの属性をファイル情報の表の情報と比較することは、更新プログラムが正しくインストールされたことを確認する方法としてサポートされていません。また、ファイル名がインストール中に変更される場合があります。ファイルまたはバージョンの情報が存在しない場合、その他の方法によって更新プログラムが正しくインストールされたことを確認してください。

-   **レジストリキーの確認**

    このセクションの「参照表」にリストされているレジストリ キーを調べることにより、セキュリティ更新プログラムがインストールしたファイルを確認することもできます。

    これらのレジストリ キーには、インストールされるファイルの完全な一覧が含まれない場合があります。また、管理者もしくは OEM メーカーによってセキュリティ更新プログラムを Windows インストール ソース ファイルに統合またはスリップストリーム化されている場合、レジストリ キーが適切に作成されない場合があります。
